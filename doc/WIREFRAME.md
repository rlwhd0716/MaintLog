## 🧭 1. 개요

본 문서는 MaintLog Android 앱의 1차 MVP 기준 주요 화면에 대한 와이어프레임을 정의한다.

목적:
- 화면 레이아웃 구조 정리
- 정보 배치 기준 정리
- 주요 액션 위치 명확화
- 화면 간 흐름 설계 보조

기준:
- 디자인 시안이 아닌 구조 중심 와이어프레임
- Android 모바일 화면 기준
- 업무 흐름 중심 구성

---

## 🏠 3. 홈 화면 (HOME)

### 목적
- 전체 업무 현황 요약
- 주요 화면으로 빠른 진입

### Wireframe

```plaintext
┌─────────────────────────────────────┐
│ MaintLog                      [🔔]  │
├─────────────────────────────────────┤
│ 업무 요약                            │
│ ┌──────────┬──────────┐            │
│ │ 요청 12건 │ 진행중 4건 │            │
│ ├──────────┼──────────┤            │
│ │ 완료 8건  │ 장애 2건   │            │
│ └──────────┴──────────┘            │
├─────────────────────────────────────┤
│ 최근 요청사항                         │
│ - 로그인 오류                  >      │
│ - API 응답 지연                >      │
│ - 사용자 권한 수정 요청         >      │
├─────────────────────────────────────┤
│ 최근 장애                            │
│ - 결제 서버 장애               >      │
│ - 파일 업로드 실패             >      │
├─────────────────────────────────────┤
│ 빠른 실행                            │
│ [요청 등록]           [장애 등록]     │
├─────────────────────────────────────┤
│ [홈] [요청] [장애] [검색] [설정]      │
└─────────────────────────────────────┘
```

### 주요 포인트
- 상단에서 앱 이름 및 공통 액션 제공
- 중간은 업무 요약 카드
- 하단은 최근 항목 + 빠른 등록 액션

## 📋 4. 요청사항 목록 화면 (REQ_LIST)

### 목적
- 요청사항 전체 조회
- 상태 필터링
- 상세 화면 진입

### Wireframe

```plaintext
┌─────────────────────────────────────┐
│ 요청사항                             │
├─────────────────────────────────────┤
│ [전체] [등록] [진행중] [완료] [보류] │
├─────────────────────────────────────┤
│ 정렬: [최신순 ▼]                     │
├─────────────────────────────────────┤
│ ┌─────────────────────────────────┐ │
│ │ 로그인 오류               진행중 │ │
│ │ 요청자: 홍길동                  │ │
│ │ 2026-04-13                      │ │
│ └─────────────────────────────────┘ │
│                                     │
│ ┌─────────────────────────────────┐ │
│ │ API 응답 지연               등록 │ │
│ │ 요청자: 김영희                  │ │
│ │ 2026-04-12                      │ │
│ └─────────────────────────────────┘ │
│                                     │
│ ┌─────────────────────────────────┐ │
│ │ 권한 수정 요청               완료 │ │
│ │ 요청자: 이철수                  │ │
│ │ 2026-04-11                      │ │
│ └─────────────────────────────────┘ │
├─────────────────────────────────────┤
│                               [+]   │
├─────────────────────────────────────┤
│ [홈] [요청] [장애] [검색] [설정]      │
└─────────────────────────────────────┘
```

### 주요 포인트
- 상태 필터를 상단에 고정
- 카드형 리스트로 가독성 확보
- FAB로 신규 등록 진입

## 📄 5. 요청사항 상세 화면 (REQ_DETAIL)

### 목적
- 요청 상세 정보 확인
- 상태 변경
- 작업 로그 조회 및 추가

### Wireframe

```plaintext
┌─────────────────────────────────────┐
│ 요청 상세                        [✎] │
├─────────────────────────────────────┤
│ 제목: 로그인 오류 발생                │
│ 상태: [진행중 ▼]                    │
│ 요청자: 홍길동                      │
│ 우선순위: 높음                      │
│ 생성일: 2026-04-13 09:30           │
├─────────────────────────────────────┤
│ 요청 내용                            │
│ ─────────────────────────────────  │
│ 로그인 버튼 클릭 시                  │
│ 앱이 종료된다는 문의 접수            │
├─────────────────────────────────────┤
│ 작업 로그                            │
│ ─────────────────────────────────  │
│ [2026-04-13 10:00]                  │
│ 상태를 진행중으로 변경               │
│ 작성자: 관리자                       │
│                                     │
│ [2026-04-13 10:30]                  │
│ 재현 테스트 진행                     │
│ 작성자: 관리자                       │
│                                     │
│                     [+ 로그 추가]    │
├─────────────────────────────────────┤
│ [홈] [요청] [장애] [검색] [설정]      │
└─────────────────────────────────────┘
```

### 주요 포인트
- 상세 화면이 핵심 업무 처리 화면
- 상태 변경과 로그 확인을 같은 화면에 배치
- 수정 버튼은 상단 우측 배치

## 📌 13. 설계 원칙

- 목록 화면은 조회 중심
- 상세 화면은 처리 중심
- 상태 변경과 로그 누적을 핵심 기능으로 설계
- 요청 / 장애 화면 구조를 최대한 통일하여 학습 비용 최소화
- 1차 MVP에서는 복잡한 시각 요소보다 정보 구조를 우선
