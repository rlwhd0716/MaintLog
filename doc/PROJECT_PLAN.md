# 📌 유지관리 업무 관리 시스템 (MaintLog)

---

## 🧭 1. 서비스 개요

### 1.1 서비스 명
- MaintLog (가칭)
- WorkTrack / OpsFlow (대체 가능)

---

### 1.2 서비스 목적

유지보수 업무는 고객 요청, 작업 처리, 장애 대응, 증적 관리 등 다양한 활동으로 구성되며,  
실제 현장에서는 이러한 정보가 메신저, 이메일, 엑셀, 문서 파일 등으로 분산되어 관리된다.

이로 인해 다음과 같은 문제가 발생한다.

- 요청사항 누락
- 처리 상태 추적 어려움
- 작업 이력 관리 비효율
- 장애 보고 문서의 검색 및 재활용 어려움

👉 본 서비스는 이러한 문제를 해결하기 위해  
**유지보수 업무의 모든 흐름을 하나의 시스템에서 통합 관리하는 것을 목표로 한다.**

---

### 1.3 타겟 사용자

- 유지보수 담당 개발자
- 운영 담당자
- PM / 관리자
- 고객 대응 담당자

---

## 💡 2. 핵심 가치

### 2.1 업무 흐름 통합 관리
- 요청 등록 → 처리 → 완료 → 이력 관리
- 모든 업무를 하나의 시스템에서 관리

---

### 2.2 상태 기반 관리
- 요청 상태를 기반으로 진행 상황 추적

---

### 2.3 증적 및 이력 관리
- 작업 로그 및 조치 내역 기록
- 이력 기반 문제 해결 및 보고 가능

---

### 2.4 멀티플랫폼 확장성
- Flutter 기반 모바일 중심 개발
- 추후 Web 확장 가능 구조 설계

---

## 📦 3. 서비스 범위

### 3.1 1차 배포 (MVP)

- 요청사항 등록 / 조회 / 수정
- 요청 상태 변경
- 작업 로그 기록
- 장애 등록 및 관리
- 검색 및 필터
- 로컬 데이터 저장

---

## 🧱 4. 서비스 구조

### 4.1 주요 도메인

- Request (요청사항)
- RequestLog (작업 로그)
- Incident (장애)
- IncidentLog (장애 조치 로그)
- User (사용자)

---

## 👤 5. 사용자 시나리오

### 5.1 요청 처리

1. 사용자는 요청사항을 등록한다  
2. 상태는 "등록"으로 설정된다  
3. 작업 시작 시 "진행중"으로 변경  
4. 작업 내용을 기록한다  
5. 완료 후 "완료" 처리  

---
